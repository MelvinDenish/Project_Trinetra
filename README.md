# Redrob — Evidence-Grounded Hybrid Ranker

Top-100 candidate ranking for the Redrob **Intelligent Candidate Discovery &
Ranking** challenge (Senior AI Engineer, Founding Team JD). A transparent,
reproducible, **CPU-only, no-network** ranking system that reasons about the gap
between *what the JD says* and *what it means* — and can be defended line-by-line
in a Stage-5 interview.

> **The one insight that drives everything:** in this dataset the `skills` array
> and `education.field_of_study` are **decoys** — assigned near-uniformly at
> random and decoupled from the candidate's real role. A ranker keyed on skills
> walks straight into the planted trap (the provided `sample_submission.csv` does
> exactly this and ranks HR Managers above ML Engineers). The real fit signal is
> in the **free-text career narrative**, so that is what we rank on.

See **[docs/PROJECT_MEMORY.md](docs/PROJECT_MEMORY.md)** for the full Phase-0→3
reasoning trail (every architecture considered and why it was eliminated) and
**[docs/DECK_OUTLINE.md](docs/DECK_OUTLINE.md)** for the presentation outline.

## How it works

```
base_fit = 0.45·S_semantic + 0.35·S_role + 0.20·S_trajectory
final    = base_fit · P_plausible · geo_factor · behavioral_factor
shortlist top-1000 by final → cross-encoder rerank → top-100 + grounded reasoning
```

- **S_semantic** — dense (BAAI/bge-small-en-v1.5) + BM25 hybrid over the candidate
  *narrative only* (headline + summary + recent role descriptions; **no skills**),
  scored against structured JD aspect-queries. Catches Tier-5 candidates who built
  recsys/search without writing the buzzword "RAG".
- **S_role** — title (a *soft* prior, never a hard filter) + domain evidence +
  experience band (derived from career history, not the noisy `yoe` field) + a
  CV/speech/robotics-only penalty.
- **S_trajectory** — rewards production/shipping evidence; bounded penalties for
  consulting-only careers, title-chasing, research-only, "AI = LangChain-only",
  and senior-with-no-recent-code.
- **P_plausible** — honeypot gate, calibrated on the full pool (≥3 expert/advanced
  skills at 0 months used; a role longer than its own dates allow). DQ guard for
  the Stage-3 honeypot check.
- **geo_factor / behavioral_factor** — bounded multipliers for location
  (no-visa-sponsorship ⇒ non-India penalized) and availability (activity, response
  rate, notice, open-to-work) — they *modulate* fit, never dominate it.
- **Cross-encoder rerank** (ms-marco-MiniLM-L-6-v2) on the ~1K shortlist sharpens
  the top-10 (50% of the composite is NDCG@10).
- **Reasoning** — deterministic evidence-ledger NLG: specific, honest, varied,
  rank-consistent, and **zero hallucination** (every fact is checked against the
  record by a CI test).

## Reproduce the submission

Requires Python 3.11–3.12 (judges' sandbox) or 3.14 (dev), CPU, 16 GB RAM.

```bash
pip install -r requirements.txt

# 1) ONE-TIME precompute (the only step that may use the network — to download
#    the embedding model — and the only step that may exceed 5 minutes).
python scripts/precompute_embeddings.py --candidates ./candidates.jsonl

# 2) RANKING STEP — CPU only, no network, <= 5 minutes. Produces the submission.
python scripts/rank.py --candidates ./candidates.jsonl --out ./submission.csv

# 3) Validate against the bundled (portal-identical) validator.
python validate_submission.py submission.csv     # -> "Submission is valid."
```

The ranking step sets `HF_HUB_OFFLINE=1` / `TRANSFORMERS_OFFLINE=1` itself, so it
runs with no network as long as the model is cached (the precompute step caches
it). Pre-computed artifacts live in `artifacts/` (git-ignored; regenerated by
step 1, or baked into the Docker image at build time — see below).

### Other commands
```bash
python scripts/profile_data.py --candidates ./candidates.jsonl   # reproduce Phase-0 numbers
python scripts/evaluate.py     --candidates ./candidates.jsonl   # anchor + ablation + honeypot report
pytest -q                                                        # unit tests
streamlit run app/streamlit_app.py                               # the demo sandbox
```

### Docker (Stage-3 reproduction container)
```bash
docker build -t redrob-ranker .          # build: downloads model + precomputes (network ok here)
docker run --rm -v "$PWD:/out" redrob-ranker \
    python scripts/rank.py --candidates ./candidates.jsonl --out /out/submission.csv
```

## Repository layout
```

  config.py        single source of truth for weights/paths/constants
  io_utils.py      streaming JSONL loader; derived-experience (resists yoe noise)
  narrative.py     candidate narrative builder (NO skills array)
  jd_spec.py       structured JD intent (aspect-queries, negatives, prefs)

  normalize.py     shared [0,1] / rank normalizers

  rerank.py        cross-encoder shortlist rerank (optional, graceful fallback)

  reasoning.py     evidence-ledger grounded NLG
  pipeline.py      orchestration (artifacts path + in-memory demo path)
scripts/           precompute_embeddings, rank, evaluate, profile_data
tests/             plausibility, scoring, validator, reasoning
app/               streamlit_app.py (sandbox demo, <=100 candidates)
docs/              PROJECT_MEMORY.md, DECK_OUTLINE.md
```

## Compute & reproducibility
- Ranking step: CPU-only, no GPU, **no network**, < 5 min (~3.5 min measured on
  100K), < 16 GB — verified by running `rank.py` with `HF_HUB_OFFLINE=1`.
- Determinism: identical input → **byte-identical CSV on a given environment**
  (verified: two runs, same SHA-256). No LLM in the scored path. Across a
  different OS/Python the *ordering* is stable but exact float bytes may differ.
- `requirements.txt` pins the **exact versions used to produce the validated CSV**
  (numpy 2.x etc.), verified on Python 3.14; the `Dockerfile` targets Python 3.11/
  Linux with the same pins (CPU torch). The Docker image build itself has not been
  run in this environment — reconcile/rebuild before relying on it. numpy
  brute-force similarity (no faiss) keeps deps light.
- Models: `BAAI/bge-small-en-v1.5` (dense), `cross-encoder/ms-marco-MiniLM-L-6-v2`
  (rerank) — both small and CPU-friendly; weights cached locally so ranking needs
  no network.
